Notification Examples

Notification Options

This page contains a set of example notifications that you can interact with to see what is possible with notifications.

Clicking on the buttons below will show one notification to demonstrate a particular options in the notification API.

If you just want to see what a notification looks like in your browser with all the UI options set, click the 'All the Options' button.

The buttons will be enabled if the browser has support for the feature. If the button is grey, then it's not supported on your browser.

Desktop & Mobile

Title & Body

Basic example of a notification with a basic title and body.

Notification Icon

The icon on a notification is the image on the left of the title and body. This is normally an image relevant to the tweet being sent.

Notification Image

Setting an image means the notification can display a large image in with the notification. At the time of writing this is behind the experimental web platform features in Chrome and the results vary between Android and Desktop.

Action Buttons

If you want to offer multiple actions to your user from the notification then you can define "actions", which add buttons to the notification that you can respond to.

Language Direction

You can set a "dir" option to indicate if the test should be displayed "left-to-right" or "right-to-left".

Tag

Defining a tag for a notification means that when a new Notification is shown with the same tag, any old notifications with that tag are removed before the new notification is shown.

Renotify

When you use the tag option, the default behavior of a new notification replacing an existing one is that there is no sound, vibration and the screen is kept asleep.

Renotify is an option that will play a sound, vibrate the device and wake up the users device when a tagged notification is replaced. This means replacing notifications is the same behavior as a completely new notification.

Silent

When a new notification arrives, it plays a sound, vibrates the users device and wakes up the screen. Setting the silent option means that the notification is displayed, but there is none of the default behaviors like sound, vibration and screen wake.

Require Interaction

On desktop a notification is only displayed for a short period of time, whereas Android devices will show the notification until the user does something with it.

To get the same behaviour on desktop and mobile you can set the "require-interaction" option to true.

Timestamp

Mobile Only

These fields aren't strictly mobile only. A desktop browser could use these fields if it want to, but at the moment they don't. For this reason, you'll need to use a mobile device to see what the options do.

Notification Badge

Notification badges are only being used on mobile, at least at the time of writing. It's used to replace the browser icon that is shown by default.

Vibration

You can define a pattern of vibrations to make a tune. This only applies to mobile devices since laptops tend not to vibrate.

Not Implemented Anywhere Yet

The following options are defined in the spec, but haven't been implemented anywhere at the time of writing. If the button is enabled, click it and see.

Sound

The sound option has been defined, but at the time of writing no browser has implemented it yet.
